# EVALUATION
Before this HW04 contribution, the SBEats project relied primarily on manual testing during development. Developers typically verified functionality by running the mobile app and interacting with the UI directly. While this allowed quick feedback during development, the project did not include any automated tests for core logic or UI components.

In particular, there was no testing framework configured in the repository, and there were no unit or component test files present. This meant that important logic such as rating aggregation, search functionality, and UI input components were not automatically validated when the code changed. As a result, regressions or edge case bugs could potentially go unnoticed.

The goal of this HW04 contribution was to introduce an automated testing setup using Jest and create a small set of unit and component tests for critical parts of the application. This provides a repeatable method for verifying functionality and improves the overall reliability of the project.
