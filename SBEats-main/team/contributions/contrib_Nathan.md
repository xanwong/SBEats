I implemented the social posts feature, which allowed users to share food photos with captions, hashtags, and an optional tagged restaurant. I built the full create-post flow from scratch — including camera and photo library access via `expo-image-picker`, base64 image encoding for Firestore storage, a restaurant search/autocomplete using the local IV restaurants dataset, and input validation to prevent submission without both a photo and a caption. I also built the post feed screen and the reusable `PostCard` component, which displays each post with the user's avatar, username, tagged location, photo, like button, caption, and hashtags, with real-time like syncing to Firestore. I then improved the posts feature by making the profile avatar, username, and tagged restaurant location on each post card interactive. Tapping the username or avatar navigates to that user's profile (or to the current user's own profile tab if the post is theirs), and tapping the restaurant location navigates to the restaurant's detail page. I also registered the `restaurant/[id]` route in the app layout to support this navigation.
