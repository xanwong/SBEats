# HW04 Scope

The planned contribution for HW04 is to implement an Instagram-style Posts feature for the SBEats app, allowing users to share food photos with captions and hashtags.

More specifically, I plan to create a dedicated Posts tab in the app where users can browse a real-time feed of food photos posted by the community. Users will also be able to create their own posts by uploading a photo, writing a caption, adding hashtags, and tagging a restaurant from the SBEats database.

# HW04 Starting Point

The current situation of the app is that users have no way to share food photos or see what others are eating. As seen in the existing app, the Discover page only shows restaurant recommendations and the Profile page only shows saved spots. There is no social feed or photo sharing functionality anywhere in the app. The only social element is the leaderboard, which ranks users by number of ratings but does not show any content from those users.

# What HW04 Will Change

After HW04, we should have 4 major changes to the app.

The first is a new **Posts tab** added to the bottom navigation bar. This gives users a dedicated place to browse the community food photo feed, sorted by most recent posts first with real-time updates via Firestore.

The second change is a **Create Post screen** accessible via the + button in the Posts tab header. Users can pick a photo from their library or take a new one, write a caption up to 300 characters, add hashtags, and optionally tag a restaurant from the SBEats restaurant database.

The third change is a **PostCard component** that displays each post in the feed with the user's avatar, username, restaurant tag, photo, like button with count, caption, and hashtags.

The fourth change is **like functionality** on posts. Any logged-in user can like or unlike a post by tapping the heart icon, and the like count updates in real time for all users via Firestore.
