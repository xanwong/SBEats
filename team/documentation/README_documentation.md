# HW04 — Documentation

## Topic

Developer-facing documentation for the SBEats codebase. A teammate is handling user/product-facing docs (README, DESIGN.md, code comments), so my contributions focus on the developer workflow and architecture side.

## Contributions

| Document | Purpose |
|----------|---------|
| `CONTRIBUTING.md` | Branching strategy, PR workflow, merge conflict resolution, and common Git mistakes with fixes |
| `ARCHITECTURE.md` | Codebase structure guide — where to put new files, how to separate screens/components/services, naming conventions, and worked examples for scaffolding new features |

## Why These Docs

Both documents address problems the team ran into during development. We had no branching strategy, which led to accidental commits on `main` and painful merge conflicts. We also had no guidance on code organization, which led to inconsistent file placement and oversized screen files. These docs give the team a shared reference to prevent those issues going forward.
